
## О подходе к разработке

Конфигурационные файлы находятся в отдельных папках `backend` и `frontend`. Так как это задание, а не крупный проект, я принял решение не использовать многие дополнительные подходы и фичи. Я могу и умею работать с фреймворками, обычно заранее продумываю архитектуру, кеширование, масштабируемость и выношу задачи по безопасности в отдельные блоки. Однако для этого ТЗ мне показалось, что это излишне, и я решил оставить все как есть.

Я не знал, нужно ли использовать фреймворки и библиотеки, так как в ТЗ этого не уточнялось. Поэтому для упрощения я сделал всё нативно.


# Простой менеджер товаров

Это приложение для учета товаров, включающее:
- **Backend**: Node.js + TypeORM + PostgreSQL
- **Frontend**: React + Vite + Tailwind CSS
- **Database**: PostgreSQL

Проект реализует базовые функции учета товаров, включая:
- Добавление, удаление и просмотр товаров
- Пагинация (50 товаров на страницу)
- Минималистичный дизайн с использованием Tailwind CSS

## Стек технологий

- **Backend**: Node.js 18+, TypeORM, PostgreSQL
- **Frontend**: React, Vite, TypeScript, Tailwind CSS
- **Database**: PostgreSQL

## Установка и запуск

### Требования

1. Убедитесь, что у вас установлен **Docker** и **Docker Compose** если вы на Linux.
   - Для установки Docker: https://docs.docker.com/get-docker/
   - Для установки Docker Compose: https://docs.docker.com/compose/install/

2. Если вы на Windows или Mac, то вам достаточно установить Docker Desktop, который включает Docker Compose.

### Шаги для запуска

1. Клонируйте репозиторий на свой компьютер:

   ```bash
   git clone https://github.com/yourusername/your-repository.git
   cd your-repository
   ```

2. Запустите контейнеры с помощью Docker Compose:

   ```bash
   docker-compose up
   ```

   - `--build`: пересобирает контейнеры перед запуском (если были изменения в Dockerfile).

3. После запуска контейнеров:
   - **Backend** будет доступен на порту **5000**. (localhost:5000)
   - **Frontend** будет доступен на порту **80**. (localhost:80)
   - **PostgreSQL** будет доступен внутри Docker-сети по хосту `db` на порту **5432**.

## Структура проекта

- **`backend/`** — Папка с серверной частью (Node.js, TypeORM, PostgreSQL).
- **`frontend/`** — Папка с клиентской частью (React, Vite, Tailwind CSS).
- **`docker-compose.yml`** — Конфигурация для запуска всех контейнеров (Backend, Frontend, Database).
- **`.gitignore`** — Файл для исключения ненужных файлов и папок из репозитория (например, `node_modules`).

## Примечания

- Миграции выполняются автоматически перед запуском сервера.
